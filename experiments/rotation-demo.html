How It Works

The Fix:

  • ‚úÖ Transform world coordinates to paper-local coordinates
  • ‚úÖ Apply inverse rotation to coordinates
  • ‚ùå NO canvas content rotation
  • ‚ùå NO texture rotation

Coordinate Transformation:

// Get local offset
localX = worldX - paperX
localZ = worldZ - paperZ

// Transform to paper-local coords
cos = Math.cos(-rotation)
sin = Math.sin(-rotation)
rotatedX = localX * cos - localZ * sin
rotatedZ = localX * sin + localZ * cos

// Map to canvas coordinates
canvasX = (rotatedX / width + 0.5) * 512
canvasY = (rotatedZ / depth + 0.5) * 512

Why this works:

  • The canvas texture is attached to the 3D paper mesh
  • When the paper rotates, the texture rotates with it
  • We just need to ensure coordinates map correctly
  • Drawing is "fixed" to the paper in paper-local space
